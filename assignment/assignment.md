# Machine Learning Project Exam Overview

---

## Final Evaluation Structure

The exam consists of two parts:

### 1. Project Component (3 deliverables)
- **Code**: Shared via GitHub (will be checked for plagiarism)
- **Written Technical Report**: Evaluated by lecturer and tutor
- **Oral Presentation**: ~10 minutes + Q&A about the project

### 2. Theory Component
- Q&A about the theory part of the course

---

## Project Rules

### Group Work
- **Maximum group size**: 2 people
- Both individuals must contribute equally to the project
- All authors must participate in the oral presentation

### Deliverables
Each group must present:
- A written technical report (will be evaluated by lecturer and tutor)
- Project code .zip shared via GitHub
- Oral presentation about (~10 min) with slides (e.g., PowerPoint or Keynote)
- Answer some questions related to the project after the presentation

---

## Report Structure

### Format Requirements
- **Length**: 8-20 pages recommended (but no limit actually)
- Must include everything listed in the structure below
- **Cover page** must include:
  - Project Title
  - Master degree
  - Course name
  - Academic Year
  - Authors
- Table of contents

### Required Sections

The following structure is the one recommended, but can be adjusted:

#### 1. MOTIVATION AND RATIONALE
- Place the project in context
- Identify research theme(s) of interest
- What problem does your project address?
- Why is it significant?

#### 2. STATE OF THE ART (SOTA) [Optional, but at least a glance]
- Describe relevant state-of-the-art methods
- What results or techniques do you plan to exploit?
- What are the weak points of SOTA methods?
- What needs to be improved? Why? How?

#### 3. OBJECTIVES
- Clearly I need to give the general and specific objectives of my project

#### 4. METHODOLOGY
- Discuss methods, algorithms, datasets used
- Describe analytical and computational tools necessary to pursue objectives
- To include:
  - Problem and task definition (classification, regression, clustering, etc.) and associated application
  - Dataset description (size, dimensionality, classes)
  - Feature extraction from data (if applied)
  - Data preprocessing (dimensionality reduction, missing values, categorical data conversion)
  - Models and their description (parameter values, how I decided on those values)

#### 5. EXPERIMENTS & RESULTS
- Describe evaluation protocol and conditions
- Datasets used
- Metrics to measure performance (accuracy, confusion matrix, recall & precision, etc.)
- Discussion on results:
  - How can results be improved in the future?
  - What are the failure cases?

#### 6. CONCLUSIONS
- Summarizing the project (goals, methods, results)
- Discussions about: (failure cases, when it performs better, where it fails)
- Imagining possible future works

#### 7. BIBLIOGRAPHY or REFERENCES
- bibliographic references relevant to the research project
- all references should correspond to a citation in the text
